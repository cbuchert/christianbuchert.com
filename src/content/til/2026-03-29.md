Designing error handling UX for functionality that requires Internet access (address autocomplete, in this case) for local-first mobile apps is challenging. Time to start poking around, see if other people have cracked this particular nut before and what their use cases were.

Local-first doesn't seem to require any real design flexibility beyond what you'd expect from a typical web app so long as you're working with your own data. Extending the app to include responsive data and functionality from external sources seems like it is a fundamental challenge to the "it feels immediate" local first model though. The obvious solution seems to be "don't let the user do a thing that requires an Internet connection if they don't have a connection." Maybe that's just the default fallback UX? Feels bad. It's hard to queue and simulate something that is big enough that you're willing to let someone else do it (like validating an address).
